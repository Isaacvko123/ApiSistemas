localidades

Ubicación física de equipos y empleados.

Índice Inicio Swagger

Descripción

Representa sitios físicos. Se usa para asignar equipos y empleados.

Relación visual rápida

Campos clave

Obligatorios
  • nombre
  • estado
Opcionales
  • codigo
  • ciudad
  • direccion
  • cp
  • notas

Rutas y permisos

RutaRolNotas
GET /localidadesAutenticadoFiltros: ?estado, ?codigo.
GET /localidades/:idAutenticadoPor id.
POST /localidadesADMIN / GERENTECrea localidad.
PATCH /localidades/:idADMIN / GERENTEActualiza datos.
DELETE /localidades/:idADMIN / GERENTEElimina registro.

Controller (métodos)

listar
  • Filtro por estado/código.
  • Orden id DESC.
obtener
  • Valida id.
  • 404 si no existe.
crear
  • Zod create.
  • Audit ENTITY_CREATE.
actualizar
  • Zod update.
  • Audit ENTITY_UPDATE.
eliminar
  • Elimina.
  • Audit ENTITY_DELETE.

Modelo / Validaciones

  • localidadCreateSchema: nombre + estado; resto opcional.
  • localidadUpdateSchema: campos opcionales.

Ejemplos (requests / responses)

POST /localidades
{
  "codigo": "GDL-01",
  "nombre": "Guadalajara",
  "estado": "Jalisco"
}
201 OK
{
  "id": 1,
  "codigo": "GDL-01",
  "nombre": "Guadalajara"
}
