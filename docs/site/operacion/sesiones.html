sesiones

Sesiones, refresh tokens y revocación.

Índice Inicio Swagger

Descripción

Tabla de sesiones activas. Se controla la vigencia del refresh token y el uso de cada sesión.

Relación visual rápida

Campos clave

Obligatorios
  • id (UUID)
  • usuarioId (FK)
  • refreshHash
  • expiresAt
Control
  • lastUsedAt
  • createdAt

Rutas relacionadas (Auth)

RutaRolNotas
POST /auth/loginPúblicoCrea sesión y tokens.
POST /auth/refreshPúblicoValida refresh y rota tokens.
POST /auth/logoutAutenticadoRevoca sesión actual.
POST /auth/logout-allAutenticadoRevoca todas las sesiones.

Flujo principal

  • login → crea sesión (refreshHash, expiración).
  • refresh → rota refreshHash y actualiza lastUsedAt.
  • logout → elimina sesión actual.
  • logout-all → elimina sesiones del usuario.

Ejemplos (requests / responses)

POST /auth/login
{
  "email": "ana@empresa.com",
  "contrasena": "MiPasswordSegura123"
}
200 OK
{
  "accessToken": "...",
  "refreshToken": "...",
  "usuario": {
    "id": 1,
    "email": "ana@empresa.com",
    "rol": "SUPERVISOR"
  }
}
POST /auth/refresh
{
  "refreshToken": "..."
}
200 OK
{
  "accessToken": "nuevo...",
  "refreshToken": "rotado..."
}

Errores comunes

HTTPCausaSolución
401Credenciales inválidasRevisar email/contraseña.
401Refresh inválido o expiradoIniciar sesión de nuevo.
403Token sin permisosRequiere token válido.

Checklist operativo

  • Validar existencia del usuario.
  • Comparar hash con Argon2.
  • Actualizar lastUsedAt en refresh.
  • Registrar auditoría en login/logout.
