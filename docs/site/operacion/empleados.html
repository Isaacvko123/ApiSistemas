empleados

Registro de empleados y sus relaciones con área, puesto y localidad.

Índice Inicio Swagger

Descripción

Representa al colaborador. Se usa en resguardos, credenciales y documentos.

Relación visual rápida

Campos clave

Obligatorios
  • nombre (2-150)
Opcionales
  • email
  • numeroEmpleado (único)
  • telefono
  • areaId, puestoId, localidadId, usuarioId
  • activo

Rutas y permisos

RutaRolNotas
GET /empleadosAutenticadoFiltros por areaId, puestoId, localidadId, activo.
GET /empleados/:idAutenticadoValida id.
POST /empleadosADMIN / GERENTEAlta con relaciones opcionales.
PATCH /empleados/:idADMIN / GERENTEActualiza y conecta/desconecta relaciones.
DELETE /empleados/:idADMIN / GERENTESoft delete (activo=false).

Controller (EmpleadoController)

listar

  • Filtros: activo, areaId, puestoId, localidadId.
  • Orden: id DESC.

obtener

  • Valida id numérico.
  • 404 si no existe.

crear

  • Valida payload con Zod.
  • Conecta relaciones si vienen ids.
  • Auditoría: ENTITY_CREATE.

actualizar

  • Permite conectar/desconectar relaciones.
  • Auditoría: ENTITY_UPDATE.

eliminar

  • Soft-delete: activo=false.
  • Auditoría: ENTITY_DELETE.

Modelo / Validaciones (EmpleadoModel)

  • empleadoCreateSchema: nombre obligatorio, resto opcional.
  • empleadoUpdateSchema: permite null para desconectar relaciones.
  • toEmpleadoCreateData: conecta área/puesto/localidad/usuario si existen.
  • toEmpleadoUpdateData: conecta/desconecta según valores.

Errores comunes

HTTPCausaSolución
400Payload inválidoRevisar campos.
404Empleado no encontradoValidar id.
409Email o número duplicadoUsar valores únicos.
409Relación inválidaVerificar ids de área/puesto/localidad.

Checklist operativo

  • Validar token y rol.
  • Revisar duplicados (email/numeroEmpleado).
  • Verificar relaciones antes de conectar.
  • Registrar auditoría en cambios.
