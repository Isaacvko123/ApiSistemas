documentos

Evidencias en PDF/imagen y enlaces a resguardos/equipos.

Índice Inicio Swagger

Descripción

Documentos asociados a equipo, resguardo o resguardo_equipo. Incluye ruta y metadatos del archivo.

Relación visual rápida

Campos clave

Obligatorios
  • tipo
  • ruta
  • Asociación a equipo/resguardo/resguardoEquipo
Opcionales
  • evento
  • mime
  • sizeBytes
  • checksum
  • empleadoId

Rutas y permisos

RutaRolNotas
GET /documentosAutenticadoFiltros: equipoId, resguardoId, resguardoEquipoId, tipo, empleadoId, evento.
GET /documentos/:idAutenticadoPor id.
POST /documentosADMIN / GERENTECrea sin archivo (ruta manual).
POST /documentos/uploadADMIN / GERENTESube archivo y crea documento.
PATCH /documentos/:idADMIN / GERENTEActualiza metadatos o relaciones.
DELETE /documentos/:idADMIN / GERENTEElimina registro.

Controller (métodos)

listar
  • Filtros múltiples.
  • Orden id DESC.
crear/subir
  • Valida con Zod.
  • Requiere relación a equipo/resguardo/resguardoEquipo.
  • Audit ENTITY_CREATE.
actualizar
  • Zod update.
  • Audit ENTITY_UPDATE.

Modelo / Validaciones

  • documentoCreateSchema: requiere relación (equipo/resguardo/resguardoEquipo).
  • documentoUpdateSchema: permite null para desconectar.

Ejemplos (requests / responses)

POST /documentos
{
  "tipo": "PDF",
  "ruta": "uploads/documentos/file.pdf",
  "resguardoId": 5
}
201 OK
{
  "id": 12,
  "tipo": "PDF",
  "resguardoId": 5
}
